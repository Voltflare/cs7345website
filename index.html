Overview: What does it do?

Sorting Made Easy:

This library allows for the creation of a user-specified vector and the running of several sorting algorithms on any vector of numbers passed into it. The vector can be specified to be in reverse order (from how it was created), completely random-shuffled (if the specified array was not already in that order), and duplicated (80% duplicated, which tests edge cases of many sorting algorithms and is thus also typically used as a benchmark).

User Input: A vector of any length containing only integers.

Sorting Algorithms supported: Bubble Sort, Merge Sort, and Insertion Sort

API Endpoints

All functions within the JS port of the library are called through a defined “Algo” object. This allows for single-file instantiation and readability of commands. The commands that can be called are divided into two families- Algo.Vector, which encompasses all functions that create and manipulate the target vector to be sorted- and Algo.Sort, which encompasses the previously mentioned sorting algorithms - Bubble, Merge, and Insertion.

There are three different modes of use for this library: Single-threaded, in which the program works and sorts as one collective unit; Multi-threaded, in which the calling program spins off a number of threads to divide the work between; and Networked, in which the calling program communicates between a central server and a number of clients to divide the work of sorting a vector. You can view the API for any of them by clicking the appropriate button next to each mode.

Single-Threaded API

Communication between the program and itself.

Multi-Threaded API

Communication between the program and a number of worker threads.

Networked Client API

Communication between a central server and several clients.

Performance Comparison

Performance testing was done by running all three algorithms (Bubble, Merge, Insertion) on a test set of three different types of data- a completely-reversed vector, a randomly-shuffled vector, and a vector where 80% of the elements are repeated. Each of these three data input types ranged from 10 elements to 100,000 elements. (Timing is done in seconds)
